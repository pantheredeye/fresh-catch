<project_specification>
<project_name>Customer Chat</project_name>

  <overview>
    Real-time 1:1 messaging between customers and Evan. Customers tap Chat in the bottom nav, enter their name (+ optional phone), and send messages. Evan sees a floating chat bubble on admin pages with unread count, opens a bottom sheet to view conversations and reply. Messages relay in real-time via per-conversation Durable Objects with WebSocket, persisted to D1 via Prisma.

    Two branches:
    - PR1 (bbb-chat-infrastructure): Schema, ChatDurableObject, WebSocket route, server functions
    - PR2 (bbb-chat-ui): Customer sheet, admin bubble/sheet/inbox, bottom nav update, unread tracking

    Target audience: older adults in north Mississippi, using phones outdoors at farmers markets. Large tap targets, high contrast, simple flows. Light mode is the primary mode — outdoor sunlight readability is critical. Use 18px as base readable font size (not 16px).

    Capacitor-ready: WebSocket reconnection, keyboard handling, haptic placeholders.

    Performance notes:
    - First message after DO hibernation has ~50-200ms cold start (Prisma init). Acceptable.
    - Message history loads as single flat query — no N+1 risk.
    - Message list uses key={msg.id} and memoization to prevent full re-renders.
    - On Capacitor foreground resume, WS reconnects and DO sends history catch-up automatically.
  </overview>
