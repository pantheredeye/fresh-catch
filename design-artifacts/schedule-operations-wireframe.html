Operations Priority Guide

Priority 1: Add Special Events
• Prominent button always visible • Quick form (name, date, time, location) • Tagged as "special" in system
Priority 2: Cancel Instances
• One-tap cancel per market • Multi-select for bulk (weather) • Clear it's just this instance
Priority 3: Context
• Day groupings for scanning • Special events visually distinct • Time info always visible
Design Decisions:
  • Cancel button always visible (no swipe gestures)
  • Select mode for bulk operations
  • Special events have visual indicator
  • Chronological list (not calendar grid)
  • No edit/reschedule in v1 (just cancel)
